# Contributing to AwesomeLinks

Thank you for considering contributing to AwesomeLinks! We appreciate your help in making this collection better.

## How to Contribute

### Reporting Issues

If you find a broken link, typo, or have suggestions for improvement:

1. Check if the issue already exists in the [Issues](../../issues) section
2. If not, create a new issue with a clear description
3. Include relevant details such as the link in question or the section affected

### Submitting Changes

#### Adding New Links

1. **Fork the repository** to your GitHub account
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/AwesomeLinks.git
   cd AwesomeLinks
   ```

3. **Create a new branch** for your changes:
   ```bash
   git checkout -b add-new-links
   ```

4. **Add your links** following these guidelines:
   - Ensure the link is relevant to the category
   - Use the format: `- [Link Title](URL) - Brief description`
   - Keep descriptions concise and informative
   - Verify that the link works and leads to quality content
   - Avoid duplicates - check if the link already exists

5. **Commit your changes**:
   ```bash
   git add README.md
   git commit -m "Add: [Link Title] to [Category]"
   ```

6. **Push to your fork**:
   ```bash
   git push origin add-new-links
   ```

7. **Create a Pull Request** from your fork to the main repository

### Pull Request Guidelines

- **One topic per PR**: Keep pull requests focused on a single topic or category
- **Clear description**: Explain what you're adding and why it's valuable
- **Check your work**: Test all links before submitting
- **Follow the format**: Maintain consistency with existing entries
- **Update the TOC**: If adding new sections, update the table of contents
